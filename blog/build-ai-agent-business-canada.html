What Is an AI Agent (and How Is It Different from a Chatbot)?

A traditional chatbot follows scripted decision trees. When a user asks something outside the script, it fails. An AI agent is fundamentally different:

  • Reasoning: It understands intent, context, and nuance — handling questions it's never seen before
  • Tool use: It can call APIs, query databases, send emails, and update CRMs — taking real actions
  • Memory: It maintains conversation context and can reference past interactions
  • Retrieval: It searches your company's knowledge base to provide accurate, sourced answers
  • Judgment: It knows when to act autonomously and when to escalate to a human
Real Example: A Halton-based HVAC company deployed our AI agent to handle service calls. The agent checks the customer's account, reviews their equipment history, suggests troubleshooting steps, and books a technician visit if needed — handling 73% of incoming calls without human intervention. Average response time dropped from 4 minutes to 12 seconds.

    max_tokens: 1024,
    system: context.systemPrompt,
    messages: context.messages,
    tools: context.availableTools,
  });
}

Data Preparation

The quality of your RAG system depends entirely on data preparation. Here's our process:

  1. Collect source documents: Product manuals, FAQs, policies, email templates, process documents
  2. Clean and normalize: Remove formatting artifacts, fix encoding, standardize terminology
  3. Chunk strategically: Split documents into 300-500 token chunks with 50-token overlap
  4. Enrich with metadata: Add source, category, date, and relevance tags to each chunk
  5. Generate embeddings: Convert chunks to vectors using a model like text-embedding-3-small
// Document chunking and embedding pipeline
import { OpenAI } from 'openai';
import { Pool } from 'pg';

const openai = new OpenAI();
const db = new Pool({ connectionString: process.env.DATABASE_URL });

interface DocumentChunk {
  content: string;
  metadata: {
    source: string;
    category: string;
    lastUpdated: string;
  };
}

async function ingestDocument(doc: DocumentChunk[]) {
  for (const chunk of doc) {
    // Generate embedding
    const embedding = await openai.embeddings.create({
      model: 'text-embedding-3-small',
      input: chunk.content,
      dimensions: 1536,
    });

    const vector = embedding.data[0].embedding;

    // Store in PostgreSQL with pgvector
    await db.query(
      `INSERT INTO document_chunks (content, embedding, metadata)
       VALUES ($1, $2::vector, $3)`,
      [chunk.content, `[${vector.join(',')}]`, JSON.stringify(chunk.metadata)]
    );
  }
}

async function searchSimilar(query: string, topK = 5) {
  const queryEmbedding = await openai.embeddings.create({
    model: 'text-embedding-3-small',
    input: query,
    dimensions: 1536,
  });

  const vector = queryEmbedding.data[0].embedding;

  const results = await db.query(
    `SELECT content, metadata, 1 - (embedding <=> $1::vector) as similarity
     FROM document_chunks
     WHERE 1 - (embedding <=> $1::vector) > 0.7
     ORDER BY similarity DESC
     LIMIT $2`,
    [`[${vector.join(',')}]`, topK]
  );

  return results.rows;
}

Step 3: Implement Tool Use

Tool use is what transforms a chatbot into an agent. Tools let your AI take real actions:

  • Database queries: Look up customer orders, inventory, account details
  • API integrations: Create tickets in Jira, update records in Salesforce, send emails via SendGrid
  • Calculations: Generate quotes, calculate shipping costs, estimate timelines
  • File operations: Generate reports, create invoices, parse uploaded documents
// Define tools for a customer service agent
const tools = [
  {
    name: "lookup_customer",
    description: "Look up a customer by email or phone number. Returns account details, recent orders, and support history.",
    input_schema: {
      type: "object",
      properties: {
        email: { type: "string", description: "Customer email address" },
        phone: { type: "string", description: "Customer phone number" },
      },
    },
  },
  {
    name: "check_order_status",
    description: "Check the current status of an order by order ID.",
    input_schema: {
      type: "object",
      properties: {
        order_id: { type: "string", description: "The order ID (e.g., ORD-12345)" },
      },
      required: ["order_id"],
    },
  },
  {
    name: "create_support_ticket",
    description: "Create a support ticket for issues that need human follow-up.",
    input_schema: {
      type: "object",
      properties: {
        customer_id: { type: "string" },
        subject: { type: "string" },
        description: { type: "string" },
        priority: { type: "string", enum: ["low", "medium", "high", "urgent"] },
      },
      required: ["customer_id", "subject", "description", "priority"],
    },
  },
  {
    name: "schedule_appointment",
    description: "Schedule a service appointment. Checks availability first.",
    input_schema: {
      type: "object",
      properties: {
        customer_id: { type: "string" },
        service_type: { type: "string" },
        preferred_date: { type: "string", description: "ISO 8601 date" },
        preferred_time: { type: "string", enum: ["morning", "afternoon", "evening"] },
      },
      required: ["customer_id", "service_type", "preferred_date"],
    },
  },
];

// Agent loop with tool execution
async function runAgent(userMessage: string, conversationHistory: any[]) {
  conversationHistory.push({ role: 'user', content: userMessage });

  while (true) {
    const response = await anthropic.messages.create({
      model: 'claude-3-5-sonnet-20241022',
      max_tokens: 1024,
      system: SYSTEM_PROMPT,
      messages: conversationHistory,
      tools,
    });

    // If the model wants to use a tool
    if (response.stop_reason === 'tool_use') {
      const toolUse = response.content.find(c => c.type === 'tool_use');
      const toolResult = await executeTool(toolUse.name, toolUse.input);

      conversationHistory.push({ role: 'assistant', content: response.content });
      conversationHistory.push({
        role: 'user',
        content: [{ type: 'tool_result', tool_use_id: toolUse.id, content: JSON.stringify(toolResult) }]
      });
      continue; // Let the model process the tool result
    }

    // Model is done — return the final text response
    conversationHistory.push({ role: 'assistant', content: response.content });
    return response.content.find(c => c.type === 'text')?.text;
  }
}

Step 4: System Prompt Engineering

Your system prompt is the agent's personality, rules, and operating manual. A well-crafted prompt is worth more than weeks of code.

const SYSTEM_PROMPT = `You are a customer service agent for [Company Name],
a [industry] company based in [City], Ontario, Canada.

## Your Role
- Help customers with inquiries about products, orders, and services
- Be professional, friendly, and concise
- Always verify customer identity before sharing account details

## Rules
1. NEVER share customer data with unauthorized parties
2. NEVER make promises about refunds or credits over $100 — escalate to human
3. NEVER generate or discuss content unrelated to [Company Name]'s products/services
4. If you're unsure about a policy, say so honestly and offer to connect with a human
5. Always confirm actions before executing them ("I'll schedule that appointment for Tuesday at 2 PM — shall I proceed?")

## Tools Available
- lookup_customer: Use when you need to find a customer's account
- check_order_status: Use when asked about an order
- create_support_ticket: Use for issues you cannot resolve directly
- schedule_appointment: Use when a customer needs a service visit

## Escalation Triggers
Immediately create a support ticket and inform the customer a human will follow up:
- Complaints about safety issues
- Legal threats or regulatory concerns
- Requests for refunds over $100
- Technical issues you cannot diagnose

## Tone
Professional but warm. Use the customer's name. Keep responses under 150 words
unless the situation requires detailed explanation.`;

Step 6: Monitor, Evaluate, and Improve

Deploying the agent is just the beginning. Continuous monitoring is critical:

Key Metrics to Track

  • Resolution rate: % of conversations resolved without human escalation (target: 70%+)
  • Hallucination rate: % of responses containing factual errors (target: <2%)
  • Average response time: Time from user message to agent response (target: <3 seconds)
  • Customer satisfaction: Post-conversation rating (target: 4.2+/5.0)
  • Cost per conversation: Total infrastructure + API costs divided by conversations (target: <$0.15)
  • Escalation quality: Were escalations appropriate? (review weekly)
// Monitoring middleware for your AI agent
async function monitoredAgentResponse(userMessage, context) {
  const startTime = Date.now();
  let response, tokensUsed, escalated;

  try {
    response = await runAgent(userMessage, context);
    tokensUsed = response.usage?.total_tokens || 0;
    escalated = response.includes('human representative');

    // Log to your monitoring system
    await logMetrics({
      timestamp: new Date().toISOString(),
      conversationId: context.id,
      responseTimeMs: Date.now() - startTime,
      tokensUsed,
      model: context.modelUsed,
      escalated,
      estimatedCost: calculateCost(tokensUsed, context.modelUsed),
    });

    return response;
  } catch (error) {
    await logError({
      conversationId: context.id,
      error: error.message,
      stack: error.stack,
    });
    return "I'm experiencing a technical issue. Let me connect you with a team member.";
  }
}

Prompt Iteration Cycle

Plan for at least 3-4 iterations of your system prompt in the first month:

  1. Week 1: Deploy with initial prompt, monitor all conversations
  2. Week 2: Review edge cases, add specific handling for common failure modes
  3. Week 3: Refine tone, add examples for tricky scenarios, tighten safety guardrails
  4. Week 4: A/B test prompt variations, measure impact on resolution rate and satisfaction

PIPEDA Compliance Checklist for AI Agents

Canadian businesses must ensure their AI agents comply with PIPEDA and provincial privacy regulations:

  • ☑️ Data minimization: Only send necessary data to the LLM — strip PII when possible
  • ☑️ Consent: Inform users they're interacting with an AI agent
  • ☑️ Data residency: Process and store data on Canadian infrastructure (AWS ca-central-1)
  • ☑️ No training on your data: Use API providers that contractually guarantee this (Anthropic, OpenAI enterprise)
  • ☑️ Audit logging: Maintain logs of all agent interactions for at least 12 months
  • ☑️ Right to access: Ability to export all data related to a specific customer
  • ☑️ Human override: Always provide an option to speak with a human
  • ☑️ Privacy Impact Assessment: Complete a PIA before processing sensitive personal information
