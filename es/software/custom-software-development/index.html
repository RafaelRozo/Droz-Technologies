§ Colaboración

Descubrimiento → Arquitectura → Construcción → Traspaso.

01 · Descubrimiento
01
Descubrimiento

Delimitación de una o dos semanas. Un socio principal se sienta con sus responsables de operaciones, TI y adquisiciones. Resultado: un resumen de una página + un mapa de partes interesadas.

02 · Arquitectura
02
Arquitectura

Documento de arquitectura de dos a cuatro semanas. Diagrama de referencia, modelo de datos, límites de servicio, modelo de amenazas, plan de secuenciación y un orden de magnitud aproximado. Revisable; firmable.

03 · Construcción
03
Construcción

Entrega iterativa en sprints de 2 semanas. Calidad de producción desde el sprint 1 (CI, pruebas, observabilidad). Demostración interna en cada sprint. Revisión de dirección trimestral.

04 · Traspaso
04
Traspaso

Documentación, runbooks, rotación de guardia, transferencia de conocimiento y una estabilización de 90 días. La colaboración se cierra o pasa a un contrato de servicio gestionado en el día 91.

§ Stack

Herramientas y tecnologías.

Lenguajes

Python (FastAPI, Django, pandas), TypeScript (Node, NestJS, tRPC), Go (cuando hay restricciones de latencia).

Python TypeScript Go FastAPI NestJS tRPC
Frontend

React, Next.js, TanStack Query; tokens del sistema de diseño consumidos desde un paquete compartido.

React Next.js TanStack Query Tokens de diseño
Datos

PostgreSQL (primario), Redis (caché, cola), Elasticsearch (búsqueda), S3 (almacenamiento de objetos), Parquet para tablas analíticas.

PostgreSQL Redis Elasticsearch S3 Parquet
Plataforma

Docker, Kubernetes o ECS Fargate, Terraform, GitHub Actions, OpenTelemetry, Sentry.
