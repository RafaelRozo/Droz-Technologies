§ Colaboración

Descubrimiento → Diseño → Construcción → Traspaso.

01 · Descubrimiento
01
Descubrimiento

Un socio principal dirige el arranque: entrevistas con usuarios, mapeo de partes interesadas, registro de hipótesis. Resultado: un resumen de producto + un mapa de recorrido.

02 · Diseño
02
Diseño

Arquitectura de información, wireframes, sistema de diseño y prototipos clicables en Figma. Revisado semanalmente con su equipo; bloqueado antes del sprint 1.

03 · Construcción
03
Construcción

Sprints de 2 semanas. Web en Next.js o React; móvil en React Native o SwiftUI. El pipeline de CI despliega a staging en cada merge; los lanzamientos a producción se controlan con una cadencia de tren de lanzamientos.

04 · Traspaso
04
Traspaso

Envío a App Store / Play Store, traspaso del alojamiento web, tableros de monitoreo, runbook de soporte y una estabilización de 90 días. Pasa a un contrato de servicio gestionado en el día 91 si es necesario.

§ Stack

Herramientas y tecnologías.

Web

Next.js, React, TypeScript, TanStack Query, Tailwind o CSS puro con tokens de diseño, Storybook.

Next.js React TypeScript TanStack Query Storybook
Móvil

React Native (multiplataforma principal), SwiftUI (cuando se requiere nativo de iOS), Kotlin / Jetpack Compose (cuando se requiere nativo de Android), Expo para iteración rápida.

React Native SwiftUI Jetpack Compose Expo
Estado y datos

Zustand, Redux Toolkit Query, Apollo, GraphQL, REST, tiempo real mediante Socket.io o WebSockets.

Zustand Apollo GraphQL WebSockets
Diseño y QA

Figma, auditorías de accesibilidad (WCAG 2.2 AA), Playwright para E2E web, Detox para E2E móvil, Sentry para errores en tiempo de ejecución.

Figma Playwright Detox Sentry
