Lo que entregamos.

01 · Landing Zones

Los primeros 90 días
de la nube bien hechos

Azure landing zones (según CAF) u organizaciones multicuenta de AWS (según AWS Well-Architected): identidad, redes, política, registro y líneas base de seguridad provisionadas como código.

CAF Well-Architected IaC
02 · Migración y rediseño de arquitectura

Re-host. Re-plataforma.
Rediseño. Retiro.

Planes de migración carga por carga: re-host, re-plataforma, rediseño de arquitectura o retiro. Piloto, ejecución en paralelo, migración, desmantelamiento — con una reversión documentada en cada paso.

Por oleadas Reversión Ejecución en paralelo
03 · Terraform / IaC

Nada se hizo a clics en una
consola tras el arranque

Toda la infraestructura como código. Módulos versionados en un registro privado. Detección de desviaciones. Pipelines de plan/apply en GitHub Actions. Nada se hizo a clics en una consola tras el arranque.

Terraform Detección de desviaciones GitHub Actions
04 · FinOps

El costo crece con la
carga, no con la plantilla

Estrategia de etiquetado, modelo de asignación, plan de capacidad reservada y una revisión de dirección trimestral. El costo en la nube crece con la carga de trabajo, no con la plantilla.

Etiquetado Asignación Capacidad reservada
§ Colaboración

Evaluación → Arquitectura → Migración → Operación.

01 · Evaluación
01
Evaluación

Evaluación de dos a cuatro semanas: inventario de cargas de trabajo, mapa de dependencias, alcance de cumplimiento (Protected-B / SOC 2) y disposición por carga (re-host, re-plataforma, rediseño, retiro). Resultado: una estrategia por escrito + ROM.

02 · Arquitectura
02
Arquitectura

Arquitectura de referencia: landing zone, modelo de identidad, topología de red, línea base de seguridad, stack de observabilidad, pipeline de despliegue y plan de recuperación ante desastres. Revisado y firmado antes de cualquier migración.

03 · Migración
03
Migración

Ejecución por oleadas. Cada oleada: piloto, ejecución en paralelo, migración, desmantelamiento. Observabilidad de calidad de producción desde la oleada 1. Dirección trimestral con el patrocinador ejecutivo.

04 · Operación
04
Operación

Opcional pero común: un contrato de servicio gestionado para operaciones en la nube, informes mensuales de FinOps, revisión trimestral de arquitectura y una revisión anual de postura.

§ Stack

Herramientas y tecnologías.

Plataformas en la nube

Microsoft Azure (principal), AWS, Google Cloud (cuando se justifica).

Microsoft Azure AWS Google Cloud
IaC y pipelines

Terraform (principal), Bicep (cuando es solo Azure y el equipo lo prefiere), GitHub Actions, Azure DevOps Pipelines, Spacelift / Terraform Cloud.

Terraform Bicep GitHub Actions Spacelift
Identidad y seguridad

Microsoft Entra ID (Azure AD), AWS IAM, KeyVault / Secrets Manager, Azure Policy, AWS SCP, Defender / GuardDuty, Sentinel / Security Hub.

Entra ID AWS IAM KeyVault Sentinel
Observabilidad

Azure Monitor + Log Analytics, AWS CloudWatch, Grafana + Prometheus, OpenTelemetry, Datadog (cuando el comprador ya lo usa).

Azure Monitor CloudWatch Grafana OpenTelemetry
