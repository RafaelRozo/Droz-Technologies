# Droz Technologies — Blog Writing Best Practices for SEO

This is NOT a blog post. This is the reference guide that MUST be followed every time a new blog post is written. The goal: every post should rank on page 1 of Google for its target keyword within Ontario.

---

## 1. KEYWORD RESEARCH (Before Writing)

**MANDATORY**: Before writing any post, use WebSearch to:
- Validate the target keyword has search volume
- Check what currently ranks for that keyword in Ontario
- Identify 3-5 secondary keywords to weave into the content
- Find 1-2 questions people ask about the topic (for FAQ schema)

**Keyword selection criteria:**
- Primary keyword: the ONE phrase you want to rank for
- Must include location modifier when relevant ("Ontario", "Canada", "Toronto", "GTA")
- Long-tail > broad (e.g., "vibration analysis for paper mills Ontario" > "vibration analysis")
- Check competitor content — find the gap they're NOT covering

---

## 2. TITLE FORMULA

**Format**: [Power Word/Number] + [Primary Keyword] + [Benefit/Location]

**Examples:**
- "5 Ways Vibration Analysis Reduces Downtime in Ontario Plants"
- "How to Win Government Software Contracts in Ontario (2026 Guide)"
- "Smart Building Retrofit ROI: What Ontario Property Owners Need to Know"

**Rules:**
- Under 60 characters (Google truncates longer titles)
- Primary keyword appears in the first half
- Include a number when possible (increases CTR 36%)
- Include year for evergreen content (shows freshness)
- Never start with "How Droz..." — lead with the keyword, not the brand

---

## 3. META DESCRIPTION

**Format**: [What the reader learns] + [Credibility signal] + [CTA]

**Rules:**
- Under 155 characters
- Include primary keyword naturally
- Include a call to action ("Learn how", "Discover", "See why")
- Don't repeat the title verbatim
- Include location when relevant ("Ontario", "Canadian")

**Example:**
"Learn how vibration analysis prevents equipment failures in Ontario manufacturing. 20+ years of field data. Free site assessment available."

---

## 4. CONTENT STRUCTURE

### Heading Hierarchy
- H1: Post title (exactly 1, auto-generated from frontmatter)
- H2: Major sections (4-8 per post)
- H3: Subsections within H2s
- Never skip levels (no H1 → H3)

### Content Length
- Cornerstone posts: 1,500-2,500 words
- Long-tail posts: 800-1,200 words
- Never pad with fluff — every sentence earns its place

### Paragraph Structure
- Max 3-4 sentences per paragraph
- Use short sentences for impact
- Open each section with the key takeaway, then explain
- Use bullet points for lists of 3+ items

---

## 5. KEYWORD DENSITY

- Primary keyword: 1-2% density (e.g., 15-30 times in a 1,500-word post)
- Include primary keyword in:
  - Title (H1)
  - First paragraph (within first 100 words)
  - At least 2 H2 headings
  - Last paragraph
  - Image alt text
  - Meta description
  - URL slug
- Secondary keywords: 0.5% each (5-10 mentions)
- NEVER keyword stuff — it must read naturally

---

## 6. INTERNAL LINKING (Minimum 3 per post)

**Required links in every post:**
1. Link to the relevant division page (e.g., PM post → /divisions/predictive-maintenance)
2. Link to at least 1 other blog post (cross-pollinate traffic)
3. Link to /contact or use "Talk to an Engineer" CTA

**Anchor text rules:**
- Use descriptive anchor text with keywords
- BAD: "click here", "learn more", "read this"
- GOOD: "our vibration analysis services in Ontario", "smart building retrofit process"
- Vary anchor text — don't use the same phrase for every link

---

## 7. EXTERNAL LINKING (1-2 per post)

- Link to authoritative sources (government sites, industry standards, research)
- Good sources: NRC, IRAP, CSA Group, ISO standards, ASHRAE, Statistics Canada
- Opens in new tab (target="_blank")
- Adds credibility and helps Google understand content context
- NEVER link to competitors

---

## 8. IMAGE OPTIMIZATION

**Every post must have:**
- 1 hero image (1200x630px for OG sharing)
- 2-4 inline images supporting the content

**Image rules:**
- Alt text MUST include primary keyword + description
  - BAD: alt="image1"
  - GOOD: alt="Vibration analysis sensor mounted on industrial turbine — predictive maintenance Ontario"
- Use WebP format when possible (smaller file size)
- Lazy load all images except hero (hero should be priority)
- Descriptive filenames: `vibration-analysis-ontario-guide.webp` not `IMG_4521.jpg`

---

## 9. URL SLUG RULES

- Lowercase, hyphens only (no underscores, no spaces)
- Include primary keyword
- Keep under 5 words when possible
- No stop words (the, and, of, for) unless they improve readability
- Example: `/blog/vibration-analysis-ontario-manufacturing`

---

## 10. SCHEMA MARKUP (Auto-generated)

Every post automatically gets:
- BlogPosting schema (headline, datePublished, author, publisher, image, description)
- BreadcrumbList (Home > Blog > Post Title)

**Ensure frontmatter is complete** — the schema is generated from it:
- `title` → headline
- `date` → datePublished
- `description` → description
- `image` → image
- `author` → author

---

## 11. ONTARIO/CANADA LOCALIZATION

**For English posts:**
- Mention "Ontario" at least 3-5 times naturally
- Reference specific cities when relevant (Burlington, Toronto, Hamilton, GTA)
- Cite Canadian regulations/standards (CSA, NRC, IRAP, PIPEDA, AODA)
- Use Canadian spelling (colour, centre, licence)
- Reference Canadian industry context (federal procurement, provincial programs)

**For French posts:**
- Target Quebec + federal government audiences
- Use Québec French, not Parisian French for industry terms
- Reference bilingual requirements in federal procurement
- Include French-specific keywords (maintenance prédictive, développement logiciel)

**For Spanish posts:**
- Target Latin American industrial markets
- Use neutral Latin American Spanish
- Reference cross-border trade opportunities (CUSMA/USMCA)

---

## 12. CTA PLACEMENT

Every post must have CTAs at these positions:
1. **After the introduction** (paragraph 2-3): Soft CTA — "Need help with [topic]? Talk to one of our engineers."
2. **Mid-article** (after a key insight): Contextual CTA related to the section content
3. **End of article**: Strong CTA — "Talk to an Engineer" button linking to /contact

**CTA rules:**
- Primary CTA: "Talk to an Engineer" (always, never "Contact Sales")
- Secondary: "Get a Free Assessment" or division-specific ("Get a Site Assessment", "Book a Demo")
- CTAs should feel helpful, not pushy
- Link to /contact with division context when possible

## 14. CONTENT QUALITY STANDARDS

**The Droz voice:**
- Authoritative but accessible — write like a senior engineer explaining to a smart VP
- Specific, not generic — real numbers, real standards, real tools
- Opinionated — take positions ("Calendar-based maintenance is dead")
- Never use: leverage, streamline, empower, robust, seamless, cutting-edge, holistic, synergy
- Use: specific dollar amounts, percentage improvements, named standards, Ontario-specific references

**Avoid AI-generated feel:**
- No "In today's rapidly evolving landscape..."
- No "It's important to note that..."
- No perfectly balanced three-point arguments
- Include one unexpected insight or contrarian take per post
- Use second person ("your plant", "your team") not third person

## 15. GEO — GENERATIVE ENGINE OPTIMIZATION (For AI Search)

AI search engines (ChatGPT, Gemini, Perplexity, Copilot) now cite content in their answers. Every post must be optimized for AI citation:

**Direct Answer Blocks:**
- Place a 40-60 word direct answer to the target query immediately after the first H2
- This answer should be self-contained — an AI can extract it without context
- 55% of AI Overview citations come from the first 30% of page content

**Citation-Friendly Structure:**
- Include specific statistics with sources (increases AI visibility by 40%)
- Use numbered lists and comparison tables (AI prefers structured data)
- Add FAQ sections that match real user prompts
- Use evidence-dense writing with specific data points

**Freshness Signals:**
- Include the current year in the title or first paragraph
- Update posts every 7-14 days when possible (citation decay starts after ~14 days)
- Add "Last updated: [date]" to frontmatter

**Schema Stacking:**
- Every post gets BlogPosting + BreadcrumbList JSON-LD (automatic)
- Add FAQPage schema for posts with FAQ sections
- Add HowTo schema for tutorial/guide posts

**Track GEO KPIs:**
- Mention Rate: % of AI answers mentioning Droz
- Citation Rate: % with clickable URLs to our content
- Position: Where we appear in the AI-generated answer

## 16. AEO — ANSWER ENGINE OPTIMIZATION (For Featured Snippets)

60% of Google searches now end without a click (answer shown on results page). Optimize for featured snippets:

**Featured Snippet Targeting:**
- Structure content for "paragraph snippets" (40-60 words answering the query)
- Use "list snippets" format (numbered steps or bullet points)
- Create "table snippets" (comparison tables with clear headers)
- Target "People Also Ask" questions with H2/H3 headings

**Answer Positioning:**
- Place the direct answer within the first 150 words of the page
- Follow with expansion, detail, and supporting evidence
- Use the exact question as an H2, then answer it directly below

**Question-Answer Format:**
- Include 3-5 "People Also Ask" style questions as H2 or H3 headings
- Answer each in 40-60 words, then expand
- Use the format: "## What is [keyword]?" → concise answer → detailed explanation

**E-E-A-T Signals:**
- Experience: Include first-hand field experience ("In our 20 years...")
- Expertise: Reference specific standards, certifications, regulations
- Authority: Cite industry bodies (ISO, CSA, NRC, IRAP)
- Trust: Include specific client results with named companies

## 17. ANTI-SLOP RULES (Stop AI-Generated Feel)

From the Stop Slop framework — every post MUST follow these rules:

**Remove filler:** Cut throat-clearing openers ("Here's the thing"), emphasis crutches ("Let that sink in"), all adverbs.

**Break formulaic structures:** No binary contrasts ("Not X. But Y."), no dramatic fragmentation, no rhetorical setups.

**Active voice always:** Every sentence needs a human subject performing an action. Name the actor.

**Be specific:** Replace "significant savings" with "$180K per avoided shutdown." No lazy extremes.

**Engage directly:** Use "you" over "people." Put readers in the scene.

**Vary rhythm:** Mix sentence lengths. Prefer two-item lists over three. No em-dashes.

**Trust readers:** State facts directly. No softening language or hand-holding justification.

**Eliminate quotables:** Rewrite anything that sounds like a pull-quote or motivational poster.

---

## 18. OGILVY COPYWRITING PRINCIPLES (For Headlines & CTAs)

**Headline formula:** Include the brand/topic + promise + benefit. Promise a specific benefit. 8-10 words for recall. Flag your audience by naming them.

**Body copy:** Long copy sells more than short. Load with facts. Show, don't tell. Add helpful information unrelated to the pitch.

**Positioning:** What does it do + who is it for? Psychological segmentation beats demographic targeting.

**The Big Idea:** Every post needs one big, simple, memorable idea. Not several mediocre ones.

---

## 19. CONTENT STRATEGY FRAMEWORK

**Searchable vs Shareable:**
- Most posts should be SEARCHABLE (match search intent with keywords)
- 1 in 5 posts should be SHAREABLE (novel insights, original data, contrarian takes)

**Buyer Journey Mapping:**
- Awareness: "what is...", "how to...", "guide to..."
- Consideration: "best...", "vs...", "comparison..."
- Decision: "pricing...", "reviews...", "demo..."

**Hub and Spoke:**
- Each division has 1 cornerstone "hub" post
- 3-5 "spoke" posts link back to the hub
- Hub posts link to all spokes
- This builds topical authority for the division's primary keyword

---

## 20. POST QUALITY SCORING (Before Publishing)

Rate each post 1-10 on these dimensions (must score 35+ out of 50 to publish):

1. **Directness** — Does it get to the point without filler?
2. **Specificity** — Does it use real numbers, names, standards?
3. **Rhythm** — Does sentence length vary naturally?
4. **Authority** — Does it demonstrate 20+ years of field experience?
5. **Density** — Does every sentence add value (no padding)?
