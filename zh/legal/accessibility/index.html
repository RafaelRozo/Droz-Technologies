无障碍声明。

Droz Technologies 致力于让所有人都能使用 droz.tech，包括使用辅助技术的人士。本页阐明我们的合规目标（WCAG 2.1 AA 级）、我们在安大略省 AODA 下的义务、我们已实施的措施，以及如何报告无障碍问题。

本声明的英文版本为具有法律效力的文本。本中文译文仅供参考之用。

1. 我们的承诺

Droz Technologies Inc. 致力于为所有用户提供一个无障碍的网站。我们设计、构建并维护 droz.tech，以达到 WCAG 2.1 AA 级，并遵守《安大略省残障人士无障碍法案（AODA），2005》，包括《综合无障碍标准条例》（IASR，O. Reg. 191/11）。

我们将无障碍视为一项制度性标准，而非一次性审计。本网站的设计系统、我们提供的局部模板，以及撰写代理清单，均在组件层面包含无障碍要求。

2. 我们以之为目标的标准

  • WCAG 2.1 AA 级——主要技术目标。Section 508（美国）和 EN 301 549（欧盟）在很大程度上与 WCAG 2.1 AA 保持一致；符合 WCAG 2.1 AA 即意味着实质上符合这些标准。
  • AODA / IASR（安大略省）——适用，因为 Droz 是一家拥有 50 名以上员工的安大略省公司，并通过加拿大政府供应商通道成为指定的公共部门供应商。
  • 魁北克省——《政府网站无障碍标准》（SGQRI 008-02）被引为 FR-CA 内容的指南。
  • 墨西哥——NMX-I-NYCE-176-2018（墨西哥无障碍标准）被引为面向墨西哥受众交付的 ES 内容的指南。

3. 我们已采取的措施

  • 语义化 HTML。每个页面都以语义方式使用 <header>、<nav>、<main>、<section>、<article>、<aside> 和 <footer>。在标记语义模糊之处应用 ARIA 角色。不使用 ARIA 标签来掩盖缺失的语义。
  • 标题层级。每个页面恰好有一个 <h1>。嵌套标题按顺序递减（<h2> → <h3> → <h4>），不跳级。
  • 键盘导航。每个交互元素（链接、按钮、表单控件、超级菜单、语言切换、移动端抽屉）均可通过键盘访问和操作。焦点通过浏览器默认样式以及表单控件上增强的轮廓线呈现可见。超级菜单在 focus-within 时与悬停时同样展开，并在按 Escape 时关闭。
  • 表单无障碍。每个表单字段都有通过 ID 关联的明确 <label for=…>；错误消息使用 aria-live="polite" 来播报验证结果；蜜罐反垃圾字段为 aria-hidden="true" 并置于屏幕外，使其不干扰辅助技术。
  • 颜色对比度。浅色背景上的正文符合 WCAG AA 对比度（正文 ≥ 4.5:1，大号显示文本 ≥ 3:1）。深色背景区块在深蓝调色板上使用白色文本，并符合 AA。
  • 替代文本。每个 <img> 都有 alt 属性。装饰性照片使用 alt=""；信息性照片（认证、客户徽标、现场摄影）带有描述性 alt 文本。
  • 焦点管理。移动端抽屉（滑出层）在打开时锁定焦点，并在关闭时将焦点返回触发元素。语言回退提示可通过键盘关闭。
  • 语言属性。每个页面均设置 <html lang="en">（或按语言设置 lang="fr" / lang="es"）。
  • 减弱动效支持。动画遵从 prefers-reduced-motion。
  • 排版与行长。在文章模板中，正文使用 17px 的 Inter Tight，行高 1.7；展示文案使用 Droz Times 并应用 text-wrap: balance；行长限制在 56–64ch。
  • 文档下载。能力说明 PDF 和媒体资料包文件均带有无障碍标签（带标题的 PDF、加注 alt 的图片）发布。尚未实现无障碍的文档会在自动回复中予以标示，并附注可应请求提供 HTML 替代版本。
  • 跳转链接。每个页面都包含一个作为首个可聚焦元素的"跳至主要内容"链接。在获得焦点前视觉隐藏。

4. 已知限制

我们已知以下限制并正在加以解决：

  • 第三方徽标。/assets/certifications/ 和 /assets/client-logos/ 中的部分认证和合作伙伴徽标以栅格 PNG/JPG/GIF 形式提供，没有矢量原件。在徽标的视觉对比度于某些背景上低于 AA 之处，其周围的说明文字承载含义（例如"机械工程师"、"加拿大机械振动"），使视觉并非信息的唯一传达方式。
  • 背景摄影。主视觉和分区横幅背景为装饰性，不承载信息；然而，在这些图像上叠加文本之处，我们保持一层渐变叠加，使叠加文本的对比度达到 AA。
  • 第 1–7 阶段期间的未翻译内容。在 FR / ES 语言镜像站点上线之前，当兄弟页面尚不存在时，语言切换会导航至该语言的主页并显示提示。这是一种过渡状态，而非稳定状态。

5. 如何报告无障碍问题

如果您在 droz.tech 上遇到无障碍障碍——辅助技术无法访问的内容、无法完成的表单、无法阅读的文档——请告知我们，以便我们修复。

  • 电子邮箱：info@droztechnologies.com（若该别名尚未启用，请使用 info@droztechnologies.com）。
  • 邮寄地址：Droz Technologies Inc., 2059 Deer Run Avenue, Burlington, Ontario, Canada——请注明 Accessibility 收。
  • 响应时间：我们在一个工作日内确认收到无障碍报告。整改时间取决于问题的性质：
    • 严重（页面对某一群体无法使用）——在 5 个工作日内修复。
    • 高（某项功能在辅助技术下无法使用）——在 10 个工作日内修复。
    • 标准（外观性或非阻断性）——在下一次计划发布中修复。

报告时，请附上页面 URL、对障碍的描述、您所使用的辅助技术或浏览器，以及（如可能）截图或屏幕录制。

6. 替代格式

droz.tech 上可供下载的文档（能力说明、媒体资料包文件、洞察 PDF、案例研究 PDF）可通过发送邮件至 info@droztechnologies.com 申请获取无障碍替代格式——大字版、纯文本、音频，或重新加注标签的 PDF。替代格式请求免费提供，并尽快办理，通常在 10 个工作日内完成。

7. 测试与评估

我们使用以下方式测试 droz.tech：(a) macOS、Windows、iOS 和 Android 上 Chrome、Firefox 和 Safari 的最新稳定版本；(b) VoiceOver、JAWS 和 NVDA 屏幕阅读器；(c) 纯键盘导航；(d) 集成到我们构建流水线中的 axe-core 自动化无障碍扫描器；(e) 对照 WCAG 2.1 AA 清单进行人工审查。

自动化和人工测试的发现项记录在公司的内部问题跟踪系统中，并在每次发布时审查。无障碍声明至少每年审查并重新发布一次，或在站点发生重大变更时更早进行。
